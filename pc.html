自由のためのPC環境

土井　翔平

2018年2月10日

このページでは備忘録としてUbuntuのデュアルブート及び仮想マシンの方法を残しています。 環境はASUS Zenbook 3 UX390UAでUbuntu GNOME 16.04.3 LTSです。

Linuxを使う

Windowsに不満を持っていない人、量的テキスト分析をする予定がない人はWindowsでいいかと思います。 Macが好きな人、嫌いではないしお金にも余裕がある人はMacでいいでしょう。 そのどちらでもない人、すなわちWindowsには不満を抱いているが、Macも好きではない人はLinuxを使うべきです。

より現実的な問題としてLinuxをどうやって使うのかというものがあります。 まず、OSの選択ですが、LinuxのおすすめOSを調べて気に入ったものを使うのが一番でしょう。 しかし、上でも触れたように、初心者で研究目的に使用するだけであればUbuntuを使うのが無難でしょう。 Ubuntuはやや癖のあるユーザーインターフェイスを用いているので、よりWindows風なOSが良ければLinux Mintも広く人気があります。 Windows風なZorinもUbuntuベースでおしゃれなので人気があると思います。

Ubuntuベースの良い点は

  1. インストールが（他のLinux系OSに比べて）簡単
  2. 操作も（他のLinux系OSに比べて）簡単
  3. ソフトウェアのインストールも（他のLinux系OSに比べて）簡単
  4. ユーザーが多いのでネット上に情報が（他のLinux系OSに比べて）多い

といったあたりでしょうか。

Ubuntuで注意しなければならないのはバージョンでしょう。 Ubuntuは開発が活発なので短期間に新しいバージョンが出てきます。 半年に一度OSを入れ直すような人は問題ないでしょうが、普通はそうではないでしょう。 したがって、5年間の保証があるLTS版をインストールすべきです。

次の問題はどのマシンを使うかです。 ごく一部の例外を除いてLinuxをOSとして搭載している製品はないかと思いますし、どのようなマシンを使うかは自由に決めたいと思います。 また、現実問題として全くWindowsやOffice製品を使わずに生きていくのは困難であると思いますので、取りうる選択肢は

  1. 仮想マシン
  2. デュアルブート

になるのではないでしょうか。

いずれの方法にせよ、使いたいOSのisoファイルをダウンロードしておいてください。

仮想マシン

仮想マシンとは、大雑把に言えば、専用のソフトウェアを用いてWindows上でLinuxを用いることです。 最大の利点は普段、Windowsでソフトを使っているように安全かつ簡単にLinuxを使える点です。 また、Linuxと同時にWindowsのソフトを使うこともできます。 逆に、問題はWindowsとLinuxを同時に用いるのでマシンのスペックを最大限活用することはできません。 したがって、Windowsと同時に使いたい人、Linuxはテキスト分析程度でメインには使わない人、PCを弄るのが怖い人は仮想マシンを使うべきでしょう。

具体的にはVirtualBoxやVMwareなどを用いることになります。 仮想マシンのソフトウェアの説明は割愛しますが、インストール方法はデュアルブートの場合と同じになります。

デュアルブート

デュアルブートとはWindowsとLInuxを別々にPCにインストールして、起動時にどちらを使用するかを選択する方法です。 メリットとデメリットは仮想マシンの場合と逆で、マシンのフルパワーを発揮できますし、（個人的体感として）Windowsに比べて電力消費が少なく、アプリケーションがさくさく動く気がします。 一方、PCの根幹部分を弄ることになるので、致命的な問題が生じないとも言えず、自己責任で行わざるを得ません。 また、タッチパッドが動かない、音量調整ができないなどハードウェアが正常に作動しない不具合が生じることもあります。 よって、スペックを最大限活用したい人、自己責任でPCをいじくれる（いじくりたい）人、可能な限りWindowsを触れたくない人はデュアルブートをおすすめしますが自己責任です。

Ubuntu系OSのインストール

USBブートローダーの作成

まず、デュアルブートをする場合に必要な設定を行います。

  1. 4GB以上の容量のUSBメモリを用意し、フォーマットします。
  2. UNetbootinやUniversal USB InstallerなどでダウンロードしたisoファイルをUSBメモリに書き込みます。
  3. コントロールパネルで高速スタートアップの機能を停止します。
  4. ディスクの管理でWindowsが入っているCドライブのボリュームを削除することでLinux用のパーティションを作成します。
    • どの程度の容量を確保すべきかは悩ましい点ですが、Linuxをメインマシンとする点、あとからパーティションを拡張するのは難しい点から、Windows側に十分な容量を確保して、残りはすべてLinux用に使うべきでしょう。
  5. 一度、PCをシャットダウンしてBIOSを起動します。
  6. BIOS画面でUSBメモリからブートします。
    • BIOSの起動方法や操作方法はメーカーや製品によって異なるので一概には言えません。

OSのインストール

ここからは仮想マシンも同様の手順になります。

  1. 4つほど選択肢が出てきますが、一番上の“Try [OS名]”のようなものを選びます。
  2. OSの体験版のようなものが起動されるので、操作感を確認します。
    • 特にデュアルブートの場合はハードウェアの致命的不具合がないかを確認します。
  3. 問題がなさそうであればデスクトップ上もしくはランチャーにある“Install [OS名]”のようなアイコンをクリックします。
  4. まず、言語を選択しますが、個人的には以下の理由より英語をおすすめします。
    • 日本語でインストールした場合、ディレクトリ名が日本語になるので、パスの不具合が生じることがないとは言えない。
      • どうしても日本語で使いたい場合は、まず英語でインストールしてあとから日本語に変更し、その際にディレクトリ名を変更しないでおくといいでしょう。
      • Linuxではターミナルと呼ばれるものでコマンドベースに操作することがありますが、その際に日本語があるとダサい。
      • また、Linuxではエラーに遭遇しないことがないと言っても過言ではないが、その際に日本語のエラーメッセージだと検索しても対処法が見つからない可能性が高いです。
      • さらに、すべてのアプリがローカライズされているとは限らず、ローカライズされていても完全に日本語化していることは少ないので、中途半端に日本語にするなら全部英語にしたほうが見栄えがいいでしょう。
  5. インストール中にアップデートするか、サードパーティのソフトをインストールするかを決めますが、どちらでもいいでしょう。
  6. 次に、OSのインストール方法を選びますが、一番上の“Install alongside with Windows”のようなものを選んでおけば（おそらく）問題はありません。
    • “Erase Windows”を選ぶとWindowsが消えますので、決別する場合以外は絶対に選ばないでください。
    • その他を選択すると自分でswap領域を決めることができます。
  7. あとは画面に従っていき、インストールが完了し、再起動すれば終了です。

これで、自由なLinuxライフが始まるわけですが、この時点では日本語入力ができないと思います。 1. Ctrl+alt+Tを押してターミナルを起動します。 - Windowsボタンを押して“terminal”と入力してもいいですが、ターミナルは頻繁に使うのでこのショートカットは覚えておくべきでしょう。 1. ターミナルで次のコマンドを入力します。

sudo apt update
sudo apt upgrade
  • これはOSやソフトウェアを更新するおまじないなので、定期的にしておきましょう。
  • sudoはスーパーユーザーとして次のコマンドを実行するという意味です。 重要なコマンドを実行する際のおまじないです。
    • sudoをつけると最初はパスワードの入力を求められるので、インストール時に設定したパスワードを入力しましょう。 キーボードを叩いてもなにも表示されませんが大丈夫なので、入力してエンターを叩きましょう。
  • もしupgradeするソフトがある場合、更新していいのか尋ねられるのでyを入力してエンターしましょう。 面倒な場合はsudo apt upgrade -yとすると自動で更新します。
  • aptコマンドはUbuntuでソフトウェアを管理する際のコマンドで頻繁に用います。
  • また、ターミナルにコマンドをペーストする際はCtrl+Shift+V、コピーする際はCtrl+Shift+Cになりますので、注意してください。

Ubuntuの初期設定

おそらく、インストールしたOSやマシンに応じていくつか問題が生じていると思います。 また、パソコンを遣いやすくするためにいくつかの設定をしておきます。 まずは、不自由なくインターネットとメールを使えるようになります。

ちなみに、Ubuntuの各バージョンやLinux mint、Zorinを一通り触った結果、Ubuntu GNOME 16.04 LTSにした理由は

  • UbuntuのAdaptaテーマとPaperアイコンがかっこいい。
  • 16.04でないとDisplaylinkを使ったUSB-HDMI接続ができなかった。
  • GNOMEだとウィンドウズの挙動に近い。

などです。

設定

まず、settingsをいじっていきます。

  • Online AcountでGoogleとMicrosoftのアカウントと同期する。
  • Region & LanguageでFormatをCanadaにする。
    • 日本だとダサく、日にち表示や温度表示が一番日本に近い英語圏がカナダなので。
  • PowerでAutomatic Suspendを有効にする。

次に、Tweak Toolをいじります。

  • ExtensionsでAlternate Tab, Application menu, Place Status Indicator, Removable drive menu, User Themeを有効にする。
  • ここでAppIndicator Supportを有効にする。
  • ここでDash to dockを有効にする。
  • ここでDynamic top barを有効にする。
  • ここでSound input & output device chooserを有効にする。
  • ここでTopicons plusを有効にする。
  • Top BarでShow Dateを有効にする。
  • WindowsでTitlebar ButtonsのMaximizeとMinimizeを有効にする。

最後に、fileを開きます。

  • トップバーのFilesの中のPreferencesでSort folders before files、Show hidden and backup filesを有効にする。
  • View new folders usingをList Viewに、Arrange itemsをBy Typeに変更する。

タッチパッド

タッチパッドをナチュラルスクロールとタップクリックに対応させるために以下のコマンドをターミナルで実行します。

gsettings set org.gnome.desktop.peripherals.touchpad natural-scroll true
gsettings set org.gnome.desktop.peripherals.touchpad tap-to-click true

日本語入力

日本語入力を可能にするために次のコマンドでfcitx-mozcをインストールします。

sudo apt install fcitx-mozc

インストールができたら、Windowsボタンを押してinput methodを検索し、fcitxを有効にします。 一度、ログアウトもしくは再起動し、タスクバーに表示されているキーボードのアイコンを左クリックします。 Configure Current Input Methodをクリックし、一覧に出ているものをすべて-ボタンで一度消します。 次に、+ボタンを押して新しく表示されたウインドウでOnly Show Current Languageのチェックを外し、検索欄で“Mozc”と入力して有効にします。 これで、日本語入力も可能になるはずです。

Bluetoothイヤホン

何もしなくてもBluttoothでイヤホンを接続することはできると思いますが、音がこもったように聞こえます。 どうやら、A2DP接続をしないといけないらしく、こちらを参考に設定します。

次のコマンドで開いたファイルにその次のコードをコピペします。 エディタは何を使ってもいいのですが、ここでは後でインストールするAtomを使っているとします。 もし、Atomをまだ入れていない場合、atomのかわりにgeditにしましょう。

sudo atom /etc/bluetooth/audio.conf
# Configuration file for the audio service
# This section contains options which are not specific to any
# particular interface
[General]
Enable=Gateway,Source

# Switch to master role for incoming connections (defaults to true)
Master=true

# If we want to disable support for specific services
# Defaults to supporting all implemented services
#Disable=Gateway,Source,Socket
Disable=Socket

# SCO routing. Either PCM or HCI (in which case audio is routed to/from ALSA)
# Defaults to HCI
#SCORouting=HCI

# Automatically connect both A2DP and HFP/HSP profiles for incoming
# connections. Some headsets that support both profiles will only connect the
# other one automatically so the default setting of true is usually a good
# idea.
AutoConnect=true

# Headset interface specific options (i.e. options which affect how the audio
# service interacts with remote headset devices)
[Headset]

# Set to true to support HFP, false means only HSP is supported
# Defaults to true
HFP=true

# Maximum number of connected HSP/HFP devices per adapter. Defaults to 1
MaxConnected=2

# Set to true to enable use of fast connectable mode (faster page scanning)
# for HFP when incoming call starts. Default settings are restored after
# call is answered or rejected. Page scan interval is much shorter and page
# scan type changed to interlaced. Such allows faster connection initiated
# by a headset.
FastConnectable=true

# Just an example of potential config options for the other interfaces
#[A2DP]
#SBCSources=1
#MPEG12Sources=0

次に、以下のコマンドで開いたファイルのload-module module-bluetooth-discoverの前に’#’を加えてコメントアウトします。

sudo atom /etc/pulse/default.pa

また、以下のコマンドでファイルを開き

sudo atom /usr/bin/start-pulseaudio-x11
if [ x"$SESSION_MANAGER" != x ] ; then
    /usr/bin/pactl load-module module-x11-xsmp "display=$DISPLAY session_manager=$SESSION_MANAGER" > /dev/null
fi

の箇所を

if [ x"$SESSION_MANAGER" != x ] ; then
    /usr/bin/pactl load-module module-x11-xsmp "display=$DISPLAY session_manager=$SESSION_MANAGER" > /dev/null
    /usr/bin/pactl load-module module-bluetooth-discover
fi

とします。

最後に、再起動し、Settingの中のSoundでBluetooth接続したイヤホンを選び、ProfiloをA2DP Sinkに変更すれば完了です。

時刻の調整

デュアルブートをした場合、Windows側の時刻が9時間ずれていることがあります。 原因はどうやらLinuxはグローバルな時刻を、Windowsはローカルな時刻を参照してるからのようです。 したがって、解決策は次の3つになります。

  1. Linux側の時計をローカルにする。 まず、次のコマンドを入力し、グローバル時間であることを確認します。 noと出ていればローカルではありません。
timedatectl status | grep local

ローカル時間にするために次のコマンドを入力します。

timedatectl set-local-rtc true

グローバル時間に戻すには上記コメンドのtrueをfalseにします。 ただし、こちらやこちらによると、ローカル時間の場合サマータイムの際などに不具合が生じるらしいです。 1. Windowsをグローバル時間にする。 まず、前述の方法でLinuxがグローバル時間であることを確認します。 タスクバーのWindowsマークを右クリックし、PowerShellを管理者として起動し、次のコマンドを入力します。

reg add HKLM\SYSTEM\CurrentControlSet\Control\TimeZoneInformation /v RealTimeIsUniversal /t REG_DWORD /d 1

しかし、こちらによると、Windowsはローカル時間前提なのでいじらないほうがいいらしいです。 1. というわけで、時間の調整を諦める、というのも選択肢かと思います。 基本的に、Windowsの時間がずれていても大きな問題はないはずですし、気づいたら直っている時もあります。 ただし、このような問題もあるらしいです。

ウェブブラウザ&メール

基本的にウェブブラウザはこれまで使ってきたものでいいかと思いますが、Windwosで標準だったInternet ExplorerやEdgeは使えません。 もし新しいブラウザを使うのだとしたらLinuxでもWindowsでもAndroidでも使えるFirefoxやGoogle Chromeがいいでしょう。

Firefoxの場合は次のコマンドでインストールできます。 多くの場合、プリインストールされています。

sudo apt install firefox

メールの場合、ブラウザで行うことが多いですが、オフラインでも使いたい場合や複数のアカウントを一括管理したい場合などはThunderbirdがいいでしょう。 多くの場合、プリインストールされています。

sudo apt install thunderbird

テーマとアイコンの変更

Linuxではテーマやアイコンを自由に選択することが可能です。 詳しくはググってほしいですが、以下では個人的な好み（マテリアルデザインかつダークテーマ）に応じたテーマとアイコンをインストールします。

まず、Adaptaというテーマをインストールします。

sudo apt-add-repository ppa:tista/adapta -y  
sudo apt update  
sudo apt install adapta-gtk-theme

次にPaperというアイコンをインストールします。

sudo apt-add-repository ppa:snwh/pulp -y
sudo apt update
sudo apt install paper-icon-theme paper-cursor-theme paper-gtk-theme

最後に、Tweak ToolのAppearanceからThemeとGnome Shell ThemeをAdapta-Nokto-Eta、IconをPaperに設定します。

Displaylink

ノートパソコンの場合、普通は外部出力は1つしかないので、デュアルディスプレイが限界ですが、USB-HDMI接続をすれば、更に外部ディスプレイを増やすことができます。 そのためには、Displaylinkのソフトをインストールして、対応する機器を使う必要があります。

ここからファイルをDownloadsというフォルダにダウントードした場合は、以下のコマンドでインストールできます。

cd ~/Downloads
sudo displaylink-debian-master/displaylink-debian.sh

それでもOfficeを使いたい

これから、Office製品と決別し、Microsoftによる支配から脱却していきます。 しかし、word, excel, power pointファイルは世界中の至るところに存在し、これらのすべてを避けて通ることは不可能に近いでしょう。 どうしてもOffice製品のファイルで作業しなければならない場合、我々には次の選択肢があります。

  1. Ubuntuに標準搭載されている無料のLibreOfficeを用いる。
  2. OneDriveもしくはDropbox上のファイルであれば、Office Onlineをブラウザ上で用いる。
  • これらの選択肢で多くのことはできますが、Officeの機能のすべてを代替することはできません。
  1. 仮想マシンの場合：Linuxの「外」でOfficeを使う。
  2. デュアルブートの場合：一度、Linuxを終了し、Windowsを起動する。

論文とスライドを作成する

論文の執筆とプレゼンテーション用のスライドの作成はLaTeXで行います。 LaTeXは数式に強いため自然科学や経済学でも用いられていますが、きれいな体裁で文書やスライドを作成できるという利点もあります。 また、beamerパッケージでスライド作成、Tikzパッケージでグラフ描画もできます。 ただ、コマンドで体裁を設定するなどやや特殊な操作なので、イニシャルコストは高いかもしれません。 それでも使い方を学ぶ価値があるとは思いますが、他にも使い道があるのでとりあえずインストールしておきましょう。

TeXLive

LaTeXは一つのソフトウェアというよりは様々なコードの集合体なので、今ひとつよく分からないですが、とりあえず使うだけであればTeXLiveをインストールすればいいと思います。

こちらを参考にしました。 なお、途中のyy, mm, ddはインストールするTeXLiveのバージョンによって違います。

cd ~/Downloads
wget http://mirror.ctan.org/systems/texlive/tlnet/install-tl-unx.tar.gz
tar xvf install-tl-unx.tar.gz
cd install-tl-20yymmdd
sudo ./install-tl --repository http://mirror.ctan.org/systems/texlive/tlnet/
sudo /usr/local/texlive/20yy/bin/x86_64-linux/tlmgr path add

かなり時間がかかりますし、途中で何故かストップして、再開しても最初からインストールしたりしますが根気よくトライしましょう。

次のコマンド（tlmgrによるTeXLiveのアップデート）を入力して何も問題がなければ成功です。

sudo tlmgr option repository http://mirror.ctan.org/systems/texlive/tlnet
sudo tlmgr update --self --all

なお、一行目のコマンドは一回実行すれば以後はする必要はありません。

TeXStudio

LaTeXは文書作成というよりは作成した文書をpdfなどに変換するプログラムのようなものです。 なので、適当なエディタで文章を書けばいいのですが、どうせなら専用のエディタを使ったほうがスムーズです。 例えば、TeXworksはシンプルなエディタとして人気です。 ここでは、もう少し機能の多いTeXstudioを使います。

sudo apt install TeXstudio

uplatexでコンパイルする場合は次のように設定します。

  • コマンドのlatexをuplatex、bibtexをupbibtex、dvipdfをdvipdfmxに書き直します。
  • ビルドでDVI -> PDFチェーンを選択し、コンパイラをLaTeXにします。

どのエンジンを使うか？

LaTeXと言っても実はいろいろな派生があります。 日本語の文章を作成する場合はuplatex+dvipdfmxを使うのがオーソドックスな気がしますが、最近はxelatexやlualatexといった次世代LaTeX日本語が出力できるそうです。 ということで、この3つを比べてみます（もう少し詳しい歴史の話はこちら）。

まず、uplatexは日本語（というかユニコード）に特化した仕様ですがdviを経由してpdfを出力するのに対して、xelatexとlualatexは直接pdfを出力しますが日本語に特化しているわけではなさそうです。 lualatexはLaTeXにLuaコードを組み込めるらしく拡張性が高いようで、今後の発展が見込めるそうですが、現在のところxelatexの方が早いらしいです。 ということで、暫定的結論は新しいxelatexを使いつつ、問題があればuplatexを使うが、lualatexの進化を注視しておく、といったところでしょうか。

LaTeXを使う

プリンストン大学政治学部のLaTeX Workshopが公開しているサンプルを使ってLaTeXが実際に動くかどうかを確認します。 とりあえず、SampleArticle.texとsample.bibをダウンロードし、TeXStudioで開きます。 画像ファイルはないので、当該箇所でエラーしないために以下の箇所を削除もしくはコメントアウトします。

  \section{Figure}
  \label{sec:fig}
  \begin{figure}[h]
   \begin{center}
    \includegraphics[scale=.7]{evdfunctions.pdf}
    \caption{The Density and Distribution Functions of Type I Extreme
    Value Distribution} \label{fig:evd}
   \end{center}
  \end{figure}

右の画面に論文らしきものが出力されれば成功です。

以下では日本語を出力できるようにします。

uplatexの場合

  1. まず、texファイルの冒頭を次のように書き換えます。
\documentclass[12pt, uplatex, dvipdfmx]{article}
  1. 次に、TeXStudioのOptions > Configure TeXStudioを開き、 commandsの中を次のように変更します。
    • LaTeX: uplatex -synctex=1 -interaction=nonstopmode %.tex
    • DviPdf: dvipdfmx %.dvi
    • BibTex: upbibtex %.aux
  2. 続いて、Buildを次のように変更します。
    • Build&View: DVI->PDF Chain
    • Default Compiler: LaTeX

xelatexの場合

  1. まず、texファイルの冒頭を次のように書き換えます。
\documentclass[12pt]{article}
\usepackage{fontspec}
\usepackage{zxjatype}
\setjamainfont{ipam.ttf}
\setjasansfont{ipag.ttf}
\setjamonofont{ipag.ttf}
\usepackage{xltxtra}
  1. また、\usepackage[pdftex, bookmarksopen=true, bookmarksnumbered=true,...のpdfte,を削除します。
  2. 続いて、Buildを次のように変更します。
    • Build&View: Compile&View
    • Default Compiler: XeLaTeX

設定を終了後、コンパイルできれば成功です。 また、Beamer_Sample.texを同様に設定し、コンパイルできればスライドを作成することもできます。 Beamerの外見に関してはここやここを参考にしてください。

バージョンを管理する

バージョン管理とはファイルの更新を記録することで、Gitと呼ばれるものを使います。 ファイルを修正しているとファイル、ファイル修正、ファイル再修正、ファイル再修正201707のようなファイルが増殖していくという悲劇に見舞われます。 Gitを使うことでよりスマートなバージョン管理を行います。

Git

Gitのインストール方法はこちらを参考にします。 ユーザー名とメールアドレスは自由に決めて構いません。 最後のコマンドは日本語名のファイルの文字化けを回避します。

sudo apt install git
git config --global user.name "User Name"
git config --global user.email "hoge@hoge.com"
git config --global core.quotepath false

Gitの使い方はまだ勉強中ですが、とりあえずこちらを参考に一人で論文のバージョン管理をできるようになります。 後述するAtomを使えばもう少し簡単にできますが、いずれにせよコマンドラインの操作を覚えていたほうがわかりやすいと思います。

フォルダの登録

Gitを使うフォルダに対して初回のみ以下の手順で登録を行います。

  1. 必要なファイルを保存するフォルダを作ります。
  2. ターミナルからそのフォルダに移動するか、そのフォルダをターミナルで開きます。
  3. git initを実行し、そのフォルダでGitの環境を作ります。

Add&commit

当該フォルダに新しいファイルができた場合、次のコマンドで状況を確認できます。

git status

次のコマンドでインデックスにaddします。

git add file-name

再び、git statusを実行するとaddされたことが分かると思います。

インデックスにあるファイルは次のコマンドでcommitされます。

git commit -m 'comment'

'comment'の中には変更内容が分かるような任意のコメントを入力します。

再び、git statusを実行すると（最新版をcommitしたので）変更が何もないことがわかります。

commitと同時にaddする場合は次のコマンドで十分です。

git commit -a -m 'comment'

もし、commitが記録に残す程のないものであれば、次のコマンドで十分です。

git commit --amend

次のようにもできます。

git commit -a --amend

履歴の確認

ファイルを編集し、git statusを実行すると再び変更が記録されていないと表示されますので、addしてcommitします。 続いて、次のコマンドを実行すると履歴が表示されます。

git log

履歴を縮小して表示する場合は、次のコマンドです。

git log --oneline

変更の確認

履歴から変更箇所を確認できます（参考）。 ファイルを変更し、addする前に前回commitしたものと比較したい場合は次のコマンドです。

git diff

addしたあとであれば、次のコマンドです。

git diff --caches

commitしたあとであれば、次のコマンドです。

git diff HEAD^

復元

過去のcommitを復元することができます。 最新のcommitに戻す場合は次のコマンドです。

git reset --hard HEAD

1つ前のcommitの場合は次のコマンドです。

git reset --hard HEAD~

2つ前のcommitの場合は次のコマンドです。

git reset --hard HEAD~2

特定のファイルだけを復元する場合は次のコマンドです。

git checkout HEAD file-name

resetもcheckoutも履歴に表示されるIDで指定することもできます。

ウェブサイトを作成する

ウェブサイトを持っている研究者は多くはないと思いますが、これから需要が高まっていく気もします。 普通はどこかのレンタルサーバを借りて、Word Pressなどで編集するかもしれませんが、ここではHTMLとCSSを自分で編集しGithub上で無料で公開する方法を紹介します。

まず、公式サイトでアカウントを作りましょう。 次に、ウェブサイトのURLを作るわけですが、Start Projectを押し、Repository Nameはuser-name.github.ioにしてください。 例えば、僕のgithub idはshohei-doiなので、このサイトのドメインはshohei-doi.github.ioになっているはずです。

続いて、SSH公開鍵を次のコマンドで取得します。

今度は、リポジトリのファルダを自分のPCに作ります。 たとえば、Documentsフォルダに作るのであれば、次のようにコマンドを入力します。

cd ~/Documents

git push -u origin master

を実行すれば終わりです。

Atomにまとめる

ひとまずここまでで、論文を書き、スライドを作り、データ分析を行い、ウェブサイトを公開するおとができるようになりました。 しかし、さらなる自由を求めてAtomでこれらの作業をすべて行いたいと思います。 その利点としては - AtomのGit機能を使える - Atomのプロジェクト機能を使える などがあります。 また、個人的経験上、LaTeXやRを異なるフロントエンドから実行できる環境を整えておくのはリスクヘッジの観点からも好ましいと思います。

ともあれ、以下ではAtomのパッケージを活用してAtom上でLaTeX, R, Pythonを動かせるようにします。 また、LaTeXよりも簡単に文書やスライドを作成できるRmarkdownについても説明します。

LaTeX

まず、AtomからLaTeXをコンパイルできるようにするために、以下のパッケージをインストールします。

  • latex：AtomからLaTeXをコンパイル
  • language-latex：LaTeX用のシンタックスハイライト
  • latexer：LaTeX用の自動補完
  • pdf-view：Atom内でpdfファイルの表示
apm install latex language-latex latexer pdf-view

.latexmkrcの編集

次に、Atomで新しいファイルを作り、以下の内容をコピペします（参考）。

#!/usr/bin/env perl
if ($^O eq 'MSWin32') {
  $latex = 'uplatex %O -kanji=utf8 -no-guess-input-enc -synctex=1 -interaction=nonstopmode %S';
  $pdflatex = 'pdflatex %O -synctex=1 -interaction=nonstopmode %S';
  $lualatex = 'lualatex -cmdx %O -synctex=1 -interaction=nonstopmode %S';
  $xelatex = 'xelatex %O -synctex=1 -interaction=nonstopmode %S';
  $biber = 'biber %O --bblencoding=utf8 -u -U --output_safechars %B';
  $bibtex = 'upbibtex %O %B';
  $makeindex = 'upmendex %O -o %D %S';
  $dvipdf = 'dvipdfmx %O -o %D %S';
  $dvips = 'dvips %O -z -f %S | convbkmk -u > %D';
  $ps2pdf = 'ps2pdf.exe %O %S %D';
  $pdf_mode = 3;
  if (-f 'C:/Program Files/SumatraPDF/SumatraPDF.exe') {
    $pdf_previewer = '"C:/Program Files/SumatraPDF/SumatraPDF.exe" -reuse-instance';
  } elsif (-f 'C:/Program Files (x86)/SumatraPDF/SumatraPDF.exe') {
    $pdf_previewer = '"C:/Program Files (x86)/SumatraPDF/SumatraPDF.exe" -reuse-instance';
  } else {
    $pdf_previewer = 'texworks';
  }
} else {
  $latex = 'uplatex %O -synctex=1 -interaction=nonstopmode %S';
  $pdflatex = 'pdflatex %O -synctex=1 -interaction=nonstopmode %S';
  $lualatex = 'lualatex %O -synctex=1 -interaction=nonstopmode %S';
  $xelatex = 'xelatex %O -synctex=1 -interaction=nonstopmode %S';
  $biber = 'biber %O --bblencoding=utf8 -u -U --output_safechars %B';
  $bibtex = 'upbibtex %O %B';
  $makeindex = 'upmendex %O -o %D %S';
  $dvipdf = 'dvipdfmx %O -o %D %S';
  $dvips = 'dvips %O -z -f %S | convbkmk -u > %D';
  $ps2pdf = 'ps2pdf %O %S %D';
  $pdf_mode = 3;
  if ($^O eq 'darwin') {
    $pvc_view_file_via_temporary = 0;
    $pdf_previewer = 'open -ga /Applications/Skim.app';
  } else {
    $pdf_previewer = 'xdg-open';
  }
}

これを、ホームディレクトリに.latexmkrcという名前で保存します（文頭のピリオドに注意）。

キーバインディングの変更

latexパッケージのビルドのショートカットキーはctrl+alt+bなのですが、僕の環境ではデジタルキーボードとかぶっていますので、TeXStudioのビルドのショートカットキーであるf1に次の手順で変更します。

  1. SettingsのKeybindingsを開く。
  2. 検索欄にctrl alt bを入力する。
  3. 一覧の中にlatex:buildとあるので、その左のコピーマークを押す。
  4. 上の方のyour keymap fileというリンクを押し、ペーストする。
  5. keymap.confを保存し、Atomを再起動する。

latexの設定

LaTeXのセクションで紹介したプリアンブルの修正に加えて、Atomのlatexパッケージの設定をこちらを参考に次のように変えます。

  1. Engineをuplatexにする。
    • 今のところ、僕の環境ではxelatexがうまくできません。
  2. Enable SyncTeXを有効にする。
  3. Output Formatをpdfにする。
  4. PDF Producerをdvipdfmxにする。
  5. Atom内で出力されたpdfファイルを見る場合はOpenerをpdf-viewにする。
    • pdfファイルをクリックするとソースファイルが該当箇所にジャンプします。

apm install markdown-pdf markdown-toc markdown-preview-enhanced

Rmarkdown

RmarkdownはRコードを実行できるmarkdownですが、謎の進化を遂げてLaTeXのようなこともできるようになっています。 残念なことに、UbuntuのRStudioは日本語環境があまり良くないので、Atomで編集し、RStudioでコンパイルするのがいいと思います。 必要なRパッケージはRStudioで最初にコンパイルするときに自動でインストールされます。

Jupyter Notebook

Jupyter Notebookは基本的にはPython用ですがRを動かすこともでき、AtomではJupyter Notebookを経由してRを動かします（参考）。 特に、Ubuntu版RStudioは日本語入力が不便なので、日本語をRで使う場合はAtomで動かしたほうがいいと思います。 しかし、ノートブック形式なので、若干RStudioとは操作感が異なります。

まず、RStudioで以下のコマンドを実行します。

install.packages{c('repr', 'IRdisplay', 'evaluate', 'crayon', 'pbdZMQ', 'devtools', 'uuid', 'digest'})
devtools::install_github('IRkernel/IRkernel')

次に、ターミナルでRを開き、次のコマンドを実行します。

IRkernel::installspec

ターミナル上でjupyter notebookを実行し、右上のnewを押してRスクリプトが対応していることを確認します。

次に、Atomで次のパッケージをインストールします。

  • language-r：R用シンタックスハイライト
  • autocomplete-r：R用自動補完
  • Hydrogen：Atom上でJupyter Kernelを使う。
apm language-r autocomplete-r hydrogen

最後に、ターミナルでjupyter kernelspec list --jsonを実行して出てきた{}で閉じられた文字列を{}を含めて「すべて」コピーし、HydrogenのsettingsのKernel Specにペーストします。

使い方自体は実行したいコードでctrl+enterを押すだけですが、RStudioと異なり、コンソールは出てきません。 なお、pythonやR以外の言語にもJupyter Notebookは対応しているのでHydrogenでも実行できると思われます。

クラウドストレージ

複数のPCを利用している人はクラウドストレージを用いてデータを同期させていることが多いと思います。 特にLinuxとWindowsの両刀使いをする場合、LinuxとWindowsの間でデータを同期させたいのは当然でしょう。

ところで、LinuxとWindowsの間でデータを共有したい時、必ずしもクラウドストレージを使う必要はありません。

  • 仮想マシンの場合：仮想マシンのソフトウェアにデータ共有フォルダを作成する機能があれば、そこに共有したいデータを保存する。
  • デュアルブートの場合：Windows側のストレージを外部ストレージとしてマウントしてアクセスする。

Linux側からWindows側のデータにアクセスできるので両方にクラウドストレージを導入して同じデータを2ヶ所で保存するのはSSD/HDDの容量が大きくない限りもったいないです。

しかし、後者の場合はリアルタイムの同期ができなかったり、パスが長くなってしまって面倒くさくなったりします。 したがって、どうしてもLinuxで同期したいデータがある場合はクラウドストレージを使うのもひとつの手だと思います。

具体的には、おなじみのDropbox,Google Drive, OneDriveに加え、MegaとownCloudを検討します。 ただし、後で説明するようにGoogle DriveとOneDriveはあまりおすすめしません。

Dropbox

DropboxはLinuxにも対応しているので、Softwareから検索してダウンロードするだけで使うことができます。 Dropboxは同期の速度も早く、Office OnlineやAcrobat Readerとの連携もいいのでおすすめです。 最大の問題は無料で使える容量が2GBと少なく、有料版だと1TBと多すぎる点です。 しかし、有料版を使うことにためらいを感じないのであれば最もいい選択肢だと思います。

MEGA

MEGAはややマイナーですが、最大の特徴は無料で50GBまで使える上に、Linuxに対応している点です。 同期の速度も操作の快適性にも特に問題は感じません。 唯一の懸念はマイナーであることで、トラブルが生じた時にネット上に情報が少ない可能性や、サービスを突然終了してしまう可能性が高いかもしれません。 といっても、サービスの終了はどのクラウドでもありうる話で、重要なデータはローカルに保存しておけば問題はないかと思います。

基本的にPC起動時に自動で同期をしてくれるはずですが、どうもMEGAが自動で起動しないようでしたら次のように設定します。 まず、Windowsボタンを押してStartup Applicationsを検索します。 一覧にMEGASyncがあるはずなので、それを選択してEditします。 commandの欄にmegasyncがあるはずなので、それをmegasync -p 15とします。 これで、起動後（やや時間が経ってから）MEGAが同期を自動で始めるはずです。

Google Drive

Google Driveは同期や操作が快適な上に無料で15GB使えるので魅力的ですが、そのままではLinuxでは使えません。 また、以下の方法で同期した場合もデータの読み込みがとても遅いので、あまり実用的ではないと判断しました。

Google DriveをLinuxで使用する方法はこちらのようにいろいろとあるそうです。 ここではひとつ目に紹介されている方法を残しておきます。

Google Driveの同期に必要なソフトウェアをインストールします。

sudo add-apt-repository ppa:alessandro-strada/ppa
sudo apt update
sudo apt install google-drive-ocamlfuse
google-drive-ocamlfuse

ブラウザ上でGoogleアカウントとパスワードを求められます。

次に、ホームディレクトリの下にGoogleDriveというフォルダを作り、そこにデータを同期させます。

mkdir ~/GoogleDrive
google-drive-ocamlfuse ~/GoogleDrive

最後に、Startup Applicationsでは以下のように設定してGoogle DriveをLinux起動時に自動マウントします。

gnome-session-properties
  • name:google-drive-ocamlfuse
  • command:google-drive-ocamlfuse /home/ユーザ名/GoogleDrive

OneDrive

OneDriveも無料で15GB使え、Office製品との連携が強力ですが、やはりLinuxではそのまま使えません。 また、OneDrive本来の問題として同期が遅い点、AndroidアプリでpdfをAcrobat Readerなどで開けない点、Officeアップロードセンターの謎のエラーが面倒な点などがあります。 加えて、以下の方法で同期した場合、（僕の環境では）ファイルそのものを編集するのではなく、コピーをして新しいファイルとしてアップロードしているという問題もありました。 したがって、OneDriveをLinuxで利用するのは非現実的であると感じます。

OneDriveをLinuxで使用する方法はこちらを参考にさせていただきました。

OneDriveの同期に必要なソフトウェアをインストールします。

sudo apt install libcurl4-openssl-dev libsqlite3-dev gdebi

次にDコンパイラ(Digital Mars D Compiler)をこちらからダウンロードします。

ホームディレクトリの下にOneDriveというフォルダを作り、いろいろ準備します。

mkdir ~/OneDrive
cd ~/OneDrive
git clone git@github.com:skilion/onedrive.git
cd onedrive
make
sudo make install
mkdir -p ~/.config/onedrive
cp ./config ~/.config/onedrive/config
systemctl --user enable onedrive
systemctl --user start onedrive
onedrive

ターミナルにURLが出てくるのでそれをブラウザで開き、Microsoftアカウントとパスワードを入力します。 真っ白な画面が現れたらそのURLをターミナルにコピペします。

最後に、Startup Applicationsでは以下のように設定してOneDriveをLinux起動時に自動マウントします。

  • name:onedrive
  • command:onedrive -m

ownCloud

ownCloudは一般的なクラウドサービスと異なり、自分でサーバーを用意します。 したがって、自分でサーバーを用意して管理する費用さえ払えば、その他は無料で容量を気にすることなく運用できます。 また、サーバー管理はLinuxでしかできないので、Linuxを使うメリットであるとも言えます。 ただし、現在勉強中です。
